# README 사용 플로우 데모 GIF (Remotion) — 설계

작성일: 2026-06-04 · 선행: [포스터 배너](2026-06-04-readme-poster-banner-design.md)

## 목표
README에 jeongyeol-finder의 **사용 플로우(검색→결과 모달)** 를 보여주는 GIF를 넣는다.
포스터 배너(정적) 아래에 배치. 코드로 만드는 영상이므로 **Remotion** 사용.
개인 토이 프로젝트라 Remotion 라이선스는 무관(사용자 확인).

## 산출물
- `remotion/` — 독립 Remotion 프로젝트(자체 package.json·node_modules는 gitignore). 배포 Next 앱 미오염.
- `public/demo.gif` — 렌더 결과(커밋).
- `README.md` — 포스터 배너 아래 `demo.gif` 삽입.

## 제작 방식
- Remotion 컴포지션을 별도 서브폴더에 둠. 앱의 디자인 토큰/구조를 재사용해 화면을 동일하게 구성하고, React state 대신 **프레임 기반 `interpolate`/`spring`** 으로 애니메이션.
  - 1차 시도: `../app/globals.css` 클래스 재사용. 고정 캔버스(1080×1350)에서 반응형 CSS가 어긋나면, 포스터에서 검증된 **토큰 기반 인라인 스타일**로 폴백(시각 결과 동일).
- 인코딩은 `@remotion/renderer` 내장 → `--codec=gif`. **시스템 ffmpeg 설치 불필요.**
- npm 의존성은 `remotion/` 안에만. 루트 Next 앱 package.json은 안 건드림(Vercel 빌드 무영향).

## 규격
- 1080×1350 (세로 4:5), 30fps, 약 9초(≈270프레임), 루프.

## 데모 예시 (실제 규정대로 — 지어내지 않음)
"예산의 변경" · 대분류 **예산운영에 관한 사항** · 기안 **담당자** · 전결권자 **국·소장** · 분기 없음(단일 답).
별표2 미니표: ★(기안)=담당자, ●(전결)=국·소장.

## 스토리보드 (30fps)
1. **0–1.2s 등장** — 글로우+그리드, "결" 브랜드, "누가 결재?" 스프링 인, 서브·검색창 페이드.
2. **1.2–3.0s 타이핑** — 검색창에 `예산의 변경` 한 글자씩 + 깜빡 커서.
3. **3.0–3.6s 검색** — "검색" 버튼 눌림 → 모달 백드롭 + 모달 슬라이드업.
4. **3.6–7.5s 결과** — 기안(담당자) · **전결권자 「국·소장」 크게** · 근거(예산운영에 관한 사항 · 예산의 변경) · 별표2 미니표.
5. **7.5–9s 홀드 → 페이드(루프)**.

## 비범위 (YAGNI)
- 실제 LLM/네트워크 호출(결과는 고정 데이터로 연출), 분기 되물음 화면, 사운드, mp4 별도본(필요시 후속).
